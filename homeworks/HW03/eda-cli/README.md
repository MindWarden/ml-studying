# S03 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

Параметры:

- `--sep` – разделитель (по умолчанию `,`);
- `--encoding` – кодировка (по умолчанию `utf-8`).

### Полный EDA-отчёт

```bash
uv run eda-cli report data/example.csv --out-dir reports
```

В результате в каталоге `reports/` появятся:

- `report.md` – основной отчёт в Markdown;
- `summary.csv` – таблица по колонкам;
- `missing.csv` – пропуски по колонкам;
- `correlation.csv` – корреляционная матрица (если есть числовые признаки);
- `top_categories/*.csv` – top-k категорий по строковым признакам;
- `hist_*.png` – гистограммы числовых колонок;
- `missing_matrix.png` – визуализация пропусков;
- `correlation_heatmap.png` – тепловая карта корреляций.

## Новые возможности команды `report` (HW03 2.3.2)

Добавлены два новых параметра командной строки:

`--title "Заголовок"`        Задаёт пользовательский заголовок отчёта (отображается как `# Заголовок` в `report.md`)
`--min-missing-share FLOAT`  Порог доли пропусков (0.0–1.0). Колонки с пропусками выше этого значения выделяются в отчёте как **проблемные** 

Эти параметры **реально используются** при генерации `report.md`:
- `--title` меняет заголовок отчёта;
- `--min-missing-share` влияет на раздел «Пропуски»: колонки с долей пропусков выше порога выводятся отдельным списком.

### Примеры вызова

```bash
# С кастомным заголовком и более строгим порогом пропусков
uv run eda-cli report data/train.csv \
    --title "EDA: Анализ данных по оттоку клиентов" \
    --min-missing-share 0.15

# Простой вызов с красивым заголовком
uv run eda-cli report data/example.csv --title "Тестовый датасет Titanic"

# С кастомным заголовком и более строгим порогом пропусков
uv run eda-cli report data/train.csv \
    --title "EDA: Анализ оттока клиентов (2025)" \
    --min-missing-share 0.15 \
    --max-hist-columns 12
